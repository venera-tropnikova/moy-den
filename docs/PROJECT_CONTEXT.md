# PROJECT_CONTEXT.md

## Название и тип продукта

**Мой день** — клиентское PWA-приложение (`manifest.json`, `display: standalone`).  
Стек: HTML / CSS / JavaScript без фреймворка.

## Главная идея

Спокойное утреннее приложение. Главный вопрос: **что меня ждёт сегодня?**  
Не ежедневник и не корпоративный планировщик.

## Для кого продукт

Для личного ежедневного открытия утром: понять день, а не сразу уйти в список дел.

## Что приложение делает

Сейчас на главной (`index.html`):

- приветствие и дата;
- погода (карточка + прогноз; данные — локальный mock в `js/weather.js`);
- праздники и даты (JSON календаря + важные даты из `localStorage`);
- историческая справка (локальный JSON, интерактивная карточка);
- открытие дня (локальный JSON, интерактивная карточка);
- поздравления (дни рождения / empty-state);
- анекдот дня;
- план на сегодня (задачи в `localStorage`).

Дополнительно есть экраны: календарь, день, дни рождения, важные даты, профиль/настройки.

## Что приложение не делает

- нет синхронизации между устройствами и облака;
- погода не ходит во внешний API (mock);
- история не подгружается из внешних «этот день в истории»;
- нет полноценного покрытия истории на все 365 дней (в JSON пока отдельные даты).

## Порядок восприятия главного экрана

Фактический порядок блоков в `index.html`:

1. Приветствие и дата  
2. Погода  
3. Праздники и даты  
4. Историческая справка + Открытие дня  
5. Поздравления + Анекдот дня  
6. План на сегодня  

Нижняя навигация: Главная · Задачи · Календарь · Настройки.

Утверждённый в паспорте идеальный порядок также включает «Мысль дня»; отдельного блока мысли на главной сейчас нет (`thoughts-ru.json` / `js/thought-of-the-day.js` в проекте есть, на `index.html` не подключены).

## Принципы продукта

Из `PROJECT_RULES.md` / паспорта:

- простота важнее количества функций;
- главный экран — про сегодняшний день;
- сначала мир вокруг человека, потом задачи;
- один этап = одна задача;
- не менять архитектуру без обсуждения.

## Источники данных (обзор)

- **Локальные JSON** в корне проекта: история, открытие дня, календарные события, мысли.
- **localStorage**: задачи, дни рождения, важные даты, настройки пользователя.
- **Внешний API погоды**: не используется (mock).
- Исторические изображения: `assets/history/`.

## Текущий статус проекта

- PWA-оболочка и главный экран работают локально (через `python -m http.server` / `Запустить Мой день.cmd`).
- History Core: карточка с closed/open/empty; база событий пока точечная, нет годового покрытия.
- Документы в `docs/` — актуальная рабочая документация.
- Корневые документы (`PROJECT_PASSPORT.md`, `PROJECT_RULES.md`, `DESIGN_PHILOSOPHY.md`, `ROADMAP.md`) пока исходные ориентиры.
