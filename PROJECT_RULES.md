# Проект «Мой день»

Перед началом любой задачи сначала полностью прочитай этот файл.

## Главная цель

Создать красивое, спокойное и удобное приложение, которое человек открывает каждый день, чтобы понять, что его ждёт сегодня.

## Принципы продукта

- Простота важнее количества функций.
- Главный экран показывает только сегодняшний день.
- Календарь — основной источник событий и задач.
- Дополнительные модули пользователь сможет включать и выключать.
- Новая функция должна приносить реальную пользу.
- Приложение должно быть внимательным, а не перегруженным.

## Работа с проектом

- Один этап = одна задача.
- Не менять файлы, не относящиеся к текущему этапу.
- Не создавать резервные копии и дубли файлов без явной просьбы.
- Не удалять файлы без подтверждения пользователя.
- Если решение можно сделать проще — предложить более простой вариант.
- Если пользователь предлагает слабое или рискованное решение — объяснить почему и предложить лучшее.
- Не соглашаться автоматически.

## После каждого этапа

1. Проверить код.
2. Исправить найденные ошибки.
3. Повторить проверку.
4. Перечислить изменённые файлы.
5. Написать, что удалось проверить автоматически.
6. Написать, что не удалось проверить автоматически.
7. Дать короткий чек-лист ручной проверки.
8. После успешных проверок запустить локальный сервер.
9. Открыть index.html через адрес локального сервера в браузере.
10. Не открывать проект через file://, если доступен локальный сервер.
11. Если автоматическое открытие браузера невозможно, написать точный адрес для ручного открытия.
12. Не завершать локальный сервер до ручной проверки пользователя, если это возможно.

## Git

Никогда не выполнять:

- git add
- git commit
- git push

Git выполняется только пользователем вручную.
