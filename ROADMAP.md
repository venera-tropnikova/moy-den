# Проект «Мой день»

## Цель проекта

«Мой день» --- приложение, которое помогает не только организовать день,
но и создать хорошее настроение с самого утра.

Главная идея --- открыть приложение и за несколько минут увидеть всё
самое важное на сегодня:

-   погоду;
-   задачи;
-   праздники;
-   историческое событие;
-   дни рождения;
-   события дня;
-   мысль дня;
-   улыбку дня.

Приложение должно быть полезным, уютным и таким, чтобы к нему хотелось
возвращаться каждый день.

------------------------------------------------------------------------

# Главный принцип разработки

Сначала --- архитектура.

Потом --- данные.

И только потом --- визуальная атмосфера.

Мы не строим красивые экраны поверх хаоса.

Мы создаём надёжные модули, которые можно развивать независимо друг от
друга.

------------------------------------------------------------------------

# Архитектура проекта

Проект состоит из независимых модулей.

## ✅ Calendar Core

Календарь.

-   государственные праздники;
-   православные праздники;
-   личные события;
-   диапазоны дат;
-   точки на календаре;
-   описания.

## ✅ History Core

Историческая справка.

Каждое событие содержит:

-   дату;
-   год;
-   название;
-   краткое описание;
-   полный текст;
-   категорию;
-   приоритет;
-   статус проверки;
-   источник.

Показываются только проверенные события.

Изображения будут добавляться позже.

## ⏳ Weather Core

Погода.

-   текущая погода;
-   прогноз;
-   красивые погодные сцены.

## ⏳ Tasks Core

Задачи.

-   список дел;
-   выполнение;
-   локальное хранение.

## ⏳ Holidays Core

Праздники.

-   государственные;
-   международные;
-   необычные.

## ⏳ Birthdays Core

Дни рождения.

-   поздравления;
-   напоминания.

## ⏳ Quotes Core

Мысль дня.

Небольшая вдохновляющая цитата.

## ⏳ Smile Core

Улыбка дня.

Добрая короткая шутка или интересный факт.

------------------------------------------------------------------------

# Историческая база

История --- отдельный долгосрочный проект.

Каждое событие должно иметь:

-   проверенную информацию;
-   ссылку на источник;
-   статус проверки.

Главная цель --- качество, а не количество.

------------------------------------------------------------------------

# Атмосфера приложения

После завершения MVP начинается следующий этап.

Главная задача --- сделать приложение живым.

Планируется:

-   погодные сцены;
-   сезонные оформления;
-   мягкие анимации;
-   единый стиль исторических иллюстраций;
-   несколько тем оформления;
-   плавная смена атмосферы.

Все эффекты должны быть лёгкими, спокойными и не отвлекать от
информации.

------------------------------------------------------------------------

# Исторические иллюстрации

Не использовать случайные изображения из интернета.

Использовать собственную библиотеку авторских иллюстраций.

Стиль:

-   современная редакционная иллюстрация;
-   единая художественная стилистика;
-   мягкие цвета;
-   премиальное оформление;
-   WebP;
-   единый размер.

Иллюстрация должна передавать смысл события, а не быть фотографией.

------------------------------------------------------------------------

# Темы оформления

В будущем пользователь сможет выбрать оформление приложения.

Например:

-   Летнее утро
-   Осенний парк
-   Зимний день
-   Весеннее цветение
-   Морской берег
-   Горное утро
-   Космос
-   Минимализм

Также будет доступен автоматический режим, когда оформление меняется в
зависимости от сезона и погоды.

------------------------------------------------------------------------

# Главная цель

Мы создаём не просто ежедневник.

Мы создаём приложение, которое хочется открыть утром.

Оно должно быть:

-   полезным;
-   красивым;
-   живым;
-   уютным;
-   современным;
-   надёжным.

Каждый новый модуль должен отвечать на вопрос:

"Сделает ли он начало дня немного лучше?"

Если ответ «да» --- значит мы движемся в правильном направлении.
