# ПРОЕКТ «МОЙ ДЕНЬ»

## Название

Мой день

## Тип приложения

PWA-приложение.

# Главная идея

«Мой день» --- спокойное утреннее приложение.

Это не ежедневник. Это не корпоративный планировщик.

Главный вопрос приложения:

> Что меня ждёт сегодня?

Приложение помогает человеку почувствовать сегодняшний день:

- погоду;
- важные даты;
- историческую справку;
- личные планы;
- маленькие приятные эмоции.

# Философия приложения

Главная ценность:

Не заставлять человека сразу заниматься делами, а помочь ему встретиться
с новым днём.

Порядок:

Мир вокруг человека → важные даты → историческая справка → личный день → задачи → маленькие приятные эмоции.

# Главный экран

Утверждённый порядок блоков:

1.  Приветствие
2.  Дата
3.  Погода
4.  Праздники и даты
5.  Историческая справка
6.  План на сегодня
7.  Дни рождения
8.  Мысль дня
9.  Анекдот дня

# Текст интерфейса

Текст должен быть коротким, лаконичным и понятным.

Правила:

- писать простыми словами;
- избегать лишних пояснений;
- не перегружать экран длинными подписями;
- оставлять только то, что помогает понять день.

# Информация дня

## Погода

В будущем: - автоматическое определение местоположения; - отображение
текущего города; - атмосфера дня.

## Праздники и даты

Правила: - актуальные даты; - проверенная информация; - краткое
описание.

## Историческая справка

Правила: - событие соответствует текущей дате; - информация
достоверная; - возможно использование изображений; - описание короткое и
интересное.

Цель: каждый день давать человеку небольшое расширение кругозора.

# Личные функции

Профиль пользователя:

- имя;
- пол: мужчина или женщина;
- дата рождения;
- местоположение: автоматически, с ручным выбором как резервом.

# Навигация

Решения:

- центральную кнопку «+» убрать;
- пункт «Ещё» заменить на настройки ⚙️;
- экран профиля и настроек должен открываться из нижней навигации.

# Анекдот дня

Правила:

- показывать до 3 анекдотов в день;
- после окончания показывать мягкое сообщение;
- не перегружать экран длинными текстами.

# Названия интерфейса

Утверждённые названия:

- «Историческая справка» вместо «Исторические события»;
- кнопка дополнительных событий: «События этого дня»;
- кнопка дат: «Важные даты этого дня».

# План на сегодня

Текущие возможности: - добавление задач; - выполнение задач; -
редактирование задач; - календарь.

# Архитектура

PWA-приложение.

Основные файлы: - index.html - calendar.html - day.html

JavaScript: - js/app.js - js/calendar.js - js/day.js

CSS: - css/style.css - css/calendar.css - css/day.css

# Хранение данных

Текущая версия:

localStorage

Ключ:

my-day-date-tasks-v1

Структура задачи:

{ id, text, done }

# Правила разработки

Одно изменение = одна задача.

После каждого изменения: - проверить код; - проверить ошибки; -
перечислить изменённые файлы; - выполнить ручной тест.

Не менять архитектуру без обсуждения.

# Будущее развитие

-   синхронизация между устройствами;
-   облачное хранение;
-   атмосферный дизайн;
-   изображения событий;
-   автоматическое определение города;
-   настройки пользователя.

# Главная цель проекта

Создать приложение, которое человек захочет открывать каждое утро.

Не просто список дел, а спокойное окно в сегодняшний день.
